Рабочая группа сопровождает  n - количество объектов (АС).
По объекту (АС) может быть несколько рабочих групп (1 линия, 2 линия, 3 линия).
Есть необходимость в рамках своей группы мониторить, маршрутизировать задачи (инциденты, ЗНО, мероприятия по риску, проблемы, ЗНИ, ЗНР, ЗПИ и т.д) в SM.
А также собирать статистику в рамках своей рабочей группы или в разрезе групп объекта.

Мониторинг заключается в следующем:
- оповещение при окончании контрольного срока задачи;
- оповещение при поступлении инцидента выше среднего;
- задачи, которые не в работе длительное время;
- оповещение при регистарции дефекта;
- Собирать задачи в группу по общей проблеме.
и т.д.

Маршрутизация задач при поступлении имеет два сценария:
Сотрудники внутри группы разделены по прикладным\функциональным темам (Пример: депозиты,  кредиты,  учетное ядро, доступы)
1) Дежурный определяет тему задачи и назначает на сотрудника (в 50% случаях тему определяет некорректно);
2) Технический пользователь (в 20% случаях тему определяет некорректно, + только в скорости назначения)
Создать логику при котором тема определяется правильно в 90% случаях.

Статистика группы:
Собирать статистику по задачам:
- нарушенные сроки;
- возвраты;
- выполненные;
- маршрутизированны на вышестоящую группу;
- маршрутизированны на смежные группы;
- маршрутизированны без анализа на вышестоящую группу;
- маршрутизированны без анализа на смежные группы;
- жалобы;
- оценки ниже 2;
- Анализ в разрезе групп по объектам;
